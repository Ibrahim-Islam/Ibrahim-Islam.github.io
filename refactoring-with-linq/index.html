Refactoring With LINQ 1

Problem Statement: You have been given a Collection of data and you have to filter the data based on a condition followed by outputting the result to console in a specified format.

Given:

class Item  
{
    public string Item1 { get; set; }
    public string Item2 { get; set; }
    public bool Condition { get; set; }
}

//Collection
var collection = new List<Item>  
{
    new Item { Item1 = "Value 1-1", Item2 = "Value 1-2", Condition = true },
    new Item { Item1 = "Value 2-1", Item2 = "Value 2-2", Condition = false },
    new Item { Item1 = "Value 3-1", Item2 = "Value 3-2", Condition = true }
};

Foreach:

StringBuilder sb = new StringBuilder();  
foreach (var item in collection)  
{
    if (item.Condition == true)
        sb.Append(String.Format("Item 1 = {0} : Item 2 = {1},", item.Item1, item.Item2));
}
var result = sb.Remove(sb.Length - 1, 1).ToString();  
Console.WriteLine(result);

As can be seen from above, looping each item from the collection and appending each item's string representation in StringBuilder followed by removing the last character which will be a comma. 8 lines.

Linq:

var result = collection  
    .Where(item => item.Condition == true)
    .Select(s => String.Format("Item 1 = {0} : Item 2 = {1}", s.Item1, s.Item2))
    .Aggregate((a, b) => a + "," + b);
Console.WriteLine(result);

Using linq, same thing is going on as above except Aggregate takes care of the extra last comma and use of StringBuilder is not required. 2 Lines.
