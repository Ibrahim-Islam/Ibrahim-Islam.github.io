Polymorphism in C#

Polymorphism is the ability of derived classes inheriting from the same base class to respond to the same method call in their own unique way.

Interface and abstract classes are ways to achieve Polymorphism in the .NET world. Polymorphism can be achieved using abstract classes and then overriding its members. Then why go for interfaces? Because .NET does not allow multiple inheritances of classes unlike languages like C++. Since, it tends to create issues like if two same named methods were inherited from two different class. It would give unexpected results as which one to invoke when called.

Using Abstract Classes

To demonstrate the use of Polymorphism and its differences I delve into code. The sample will be about types of Bank Accounts being inherited from Account and how same call can be used to invoke individual behavior.

//Account.cs
abstract class Account  
{
    public abstract string GetAccountInfo();
}


//SavingsAccount.cs
class SavingsAccount : Account  
{
  public override string GetAccountInfo()
  {
    return "This account information is from Savings Account";
  }
}


//program.cs
class Program  
{
  static void Main(string[] args)
  {
      CheckingAccount oCheckingAccount = new CheckingAccount();
      SavingsAccount oSavingsAccount = new SavingsAccount();

      PrintAccountInfo(oCheckingAccount);
      PrintAccountInfo(oSavingsAccount);

      Console.ReadKey();
  }   
  static void PrintAccountInfo(Account account)
  {
      Console.WriteLine(account.GetAccountInfo());
  }
}

I have an abstract class above whose member implementation are done on the derived classes. So, 2 different method implementations but with same name inherited from the same type. When I attempt to use it after initialization of each account types, I do not access their members using their objects instead I use Account's GetAccountInfo() to get my results. Thus, same method call but different response for each types of bank account which is consistent with the definition I provided above. You might wonder how we could pass objects of type CheckingAccount and SavingsAccount where the parameter type is of Account. Well, it is possible because CheckingAcount and SavingsAccount inherited from Account thus, also being a type of Account and can be assigned to an object of type Account.

Using Interfaces

//Account.cs
interface IAccount  
{
    string GetAccountInfo();
}


//SavingsAccount.cs
class SavingsAccount : IAccount  
{
  public string GetAccountInfo()
  {
    return "This account information is from Savings Account";
  }
}


//CheckingAccount.cs
class CheckingAccount : IAccount  
{
  public string GetAccountInfo()
  {
      return "This account information is from Checking Account";
  }
}


//program.cs
class Program  
{
  static void Main(string[] args)
  {
    CheckingAccount oCheckingAccount = new CheckingAccount();
    SavingsAccount oSavingsAccount = new SavingsAccount();

    PrintAccountInfo(oCheckingAccount);
    PrintAccountInfo(oSavingsAccount);

    Console.ReadKey();
  }    

  static void PrintAccountInfo(IAccount account)
  {
    Console.WriteLine(account.GetAccountInfo());
  }
}

Same process as with Abstract classes done here with Interfaces. Instead of type Account we inherited from an interface IAccount and accessing it methods instead of individual methods from bank account type objects. Some features of Interface narrated below:

  • Cannot contain Fields but Properties allowed
  • Interfaces cannot have method definitions. Only declerations e.g. Method signature.
  • If interface inherited then ALL members are to be implemented including any member down the inheritance chain
  • Objects of the class that implemented the interface is equal to the interface (type ) as well
  • A class can implement multiple interfaces
  • Interfaces support Inheritance
  • Provides a roadmap/rules to how class implementation can be changed
  • Interfaces are only limited to Access Modifiers of public
